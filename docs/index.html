    Introduction

    Welcome to the Forio Epicenter API!

    Authentication

    Example Request

    curl -X POST \
  https://test.forio.com/v3/epicenter/manager/authentication \
  --header 'Content-Type: application/json' \
  --data '{
  "handle": "john_doe@forio.com",
  "password": "<your account password>",
  "objectType": "user"
}

    Example Response

    {
  "session": "eyJ0eXAiOiJKV1QiLA0K.....",
  "timestamp": "2019-03-10T18:28:51.389Z",
  "expires": true,
  "timeoutMinutes": 240,
  "possibleTeamAccountShortNames": [
      "lakers",
      "warriors"
  ],
  "personalAccountShortName": "john_doe",
  "userKey": "000000000000000000000000000000000000",
  "userHandle": "john_doe@forio.com",
  "objectType": "user"
}

    Authentication to the Epicenter API is done by making a request to the authentication endpoint and providing a valid authentication token (a JSON object) as part of the request body. Once authenticated, you'll be granted with a session-specific access token that you can then use to call any of our other APIs successfully.

    Our API provides five types of access tokens:

    • User access tokens are specific to an Epicenter author, team member, or end user. API calls using this kind of token have access to all of the data that the user has access to.

    • Player access tokens

    • Project access tokens are created with the Public and Secret API keys for the project. API calls using this kind of token have access to all of the data in the project.

    • Admin access tokens

    • Anonymous access tokens

    To authenticate, provide these settings as part of your request:

    HTTP Request
    POST /v3/epicenter/manager/authentication

    Headers
    Set Content-Type to application/json

    Example request body
    { "handle": "john_doe@forio.com", "password": "<your account password>", "objectType": "user" }

    A successful request returns a polymorphic JSON object based on the type of authentication token you've provided.

    As an example, displayed at the right is how to create a user access token. It shows the parameters you need to include in your request body and the expected response object to be returned.

    The following sections below show the various methods of creating the five types of access tokens:

    • Creating a user access token
    • Creating a player access token
    • Creating a project access token
    • Creating an anonymous access token
    • Creating an admin access token
    These various methods use the same request method and authentication endpoint as described previously. They differ only on the JSON payload you include on your request body.

    Creating a user access token

    Example Request

    curl -X POST \
  https://test.forio.com/v3/epicenter/manager/authentication \
  -H 'Content-Type: application/json' \
  -d '{
  "handle": "john_doe@forio.com",
  "password": "<your account password>",
  "objectType": "user",
  "teamAccountShortName": "lakers"
}

    Example Response

    {
    "session": "eyJ0eXAiOiJKV1QiLA0K.....",
    "timestamp": "2019-03-10T19:21:33.721Z",
    "expires": true,
    "timeoutMinutes": 240,
    "teamAccountRole": "AUTHOR",
    "personalAccountShortName": "john_doe",
    "teamAccountShortName": "lakers",
    "userKey": "000000000000000000000000000000000000",
    "userHandle": "john_doe@forio.com",
    "objectType": "user"
}

    Request Body Parameters

    Required parameters

    Name /type Description /example
    handle
    string    		 Your account's email.
    password
    string    		 You account's password.
    objectType
    const    		 The string value "user".

    Optional parameters

    Name /type Description /example
    teamAccountShortName
    string    		 The short name of a ‘team’ account, i.e. a paying licensed account which may have more than one author (team member).
    If you don't include this in your request, then all teams under the user are shown in the possibleTeamAccountShortNames attribute as part of the response.

    Returns

    Returns a UserWhoAmI object.

    The UserWhoAmI object

    Name /type Description /example
    userHandle
    string    		 The user’s chosen login name. It's usually your account's email.
    personalAccountShortName
    string    		 The short name of a ‘personal’ account, i.e., a free account with limitation provided to each person who signs up for Epicenter.
    expires
    boolean    		 true if the token/session has an expiry. Otherwise, false.
    session
    string    		 The generated specific access token.
    teamAccountRole
    enum    		 The ‘role’ of an author on an account, which is either "AUTHOR" (a full team member) or "SUPPORT" (someone with lesser privileges in a customer support role).
    possibleTeamAccountShortNames
    array (string)    		 A list of account shortNames to which the user has rights (has roles for). Another authorization request should be sent with one of these accounts chosen.
    teamAccountShortName
    string    		 The short name of a team account, i.e. a paying licensed account which may have more than one author (team member).
    timeoutMinutes
    number    		 The number of minutes until the token expires.
    userKey
    string    		 The 36 character unique identifier for a user.
    timestamp
    string (DateTime)    		 The time that the request was made.
    objectType
    const    		 "user"

    Creating a project access token

    Example Request

    curl -X POST \
  https://test.forio.com/v3/epicenter/manager/authentication \
  -H 'Content-Type: application/json' \
  -d '{
  "secretKey": "<your project's secret key>",
  "projectKey": "<your project's public key>",
  "objectType": "project"
}

    Example Response

    {
    "session": "eyJ0eXAiOiJKV1QiLA0K.....",
    "timestamp": "2019-03-10T22:54:02.314Z",
    "expires": false,
    "timeoutMinutes": 240,
    "accountShortName": "lakers",
    "objectType": "project"
}

    Request Body Parameters

    Required parameters

    Name /type Description /example
    projectKey
    string    		 The public key of a project. It's a part of the pair of API keys generated for each project.
    secretKey
    string    		 The secret key of a project. It's the matching pair of API keys generated for each project, and is needed together with the public key to generate an access token.
    objectType
    const    		 The string value "project".

    Returns

    Returns a ProjectWhoAmI object.

    The ProjectWhoAmI object

    Name /type Description /example
    expires
    boolean    		 true if the token/session has an expiry. Otherwise, false.
    accountShortName
    string
    session
    string    		 The generated specific access token.
    timeoutMinutes
    number    		 The number of minutes until the token expires.
    timestamp
    string (datetime)    		 The time that the request was made.
    objectType
    const    		 "project"
